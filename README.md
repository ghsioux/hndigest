# Hacker News Digest

A GitHub Pages static site that publishes a daily digest of the hottest Hacker News
tech / computer-science stories, tools, and discussions. The digest is generated by an
agentic workflow: an AI agent runs on a daily schedule, reads the Hacker News front
page, and curates and summarizes the top technical stories in natural language. At the
end of each run the site is updated automatically with that day's digest. It features a
calendar sidebar and a Markdown-rendered digest pane, in a Hacker News inspired theme.

## How it works

The site is fully static. There is no build step and no backend:

- **`index.html`** - single-page app. Renders a month calendar, highlights days that have
  a digest, and loads the selected day's Markdown with [marked](https://github.com/markedjs/marked).
- **`digests/manifest.json`** - a JSON array of available digest dates (`YYYY-MM-DD`).
  The calendar reads this to know which days are clickable.
- **`digests/YYYY-MM-DD.md`** - one Markdown file per day. The app fetches and renders it.
- **`.nojekyll`** - disables Jekyll so the raw `.md` files are served verbatim.

## Digest format

Each digest is Markdown with a stable shape (sections are optional and counts vary per day):

```
# Hacker News Digest -- YYYY-MM-DD
---
## Top Stories          (3–5 items: ### [Title](hn-link) + 1 short summary paragraph)
## Projects & Tools     (bulleted: **[Name](hn-link)** -- one-liner, optional (repo) link)
## Worth Reading        (bulleted links + one-liners)
## Community Pulse       (optional: Ask HN / discussion highlights)
---
*Generated from [Hacker News](https://news.ycombinator.com) top stories and discussions.*
```

## Adding a new digest

1. Write `digests/YYYY-MM-DD.md` following the format above.
2. Append the date to `digests/manifest.json` (keep it sorted).

## How content is stored and published

Each day's digest is stored as Markdown directly in this repository:

- **`digests/YYYY-MM-DD.md`** - the digest content for that day.
- **`digests/manifest.json`** - the sorted list of dates that have a digest.

The daily agentic generator lives in `.github/workflows/hacker-news-digest.md` (compiled
to `.github/workflows/hacker-news-digest.lock.yml`). It runs every day at 08:13
Europe/Paris and can also be triggered manually with:

```bash
gh aw run hacker-news-digest
```

The agent reads the Hacker News front page and Show HN, summarizes the technical stories
into the Markdown format above, and writes `digests/<date>.md` plus the manifest entry.
For safety the agent itself never writes to the main branch directly: it proposes the new
digest, the `Publish Digest` workflow merges it automatically, and the `Deploy GitHub
Pages` workflow republishes the site. The net result is that the site shows the new digest
shortly after each run finishes, with no manual step. If a digest already exists for the
day, the run is a no-op.
